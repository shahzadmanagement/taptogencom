## Performance And UX

- The generic tool page script is very large: `src/pages/tools/[slug].astro` is about `395 KB`, and `src/data/tools.ts` is about `463 KB`. This is likely the biggest front-end performance risk because every tool page carries the full switch-based generator logic.
- Build output succeeded, but cleaning `dist` can hit Windows/OneDrive permission locks. CI or local launch builds should run from an unlocked workspace or after closing processes watching `dist`.
- SVG/image risk is currently limited to `fantasy-map-generator`; no external image dependency issues found.
- Mobile responsiveness risk is moderate because the generic output area can receive very long text/SVG/pre blocks. The Fantasy Map implementation uses overflow handling, but older tools may still produce long unwrapped text.
- Search/filter usability exists on the tools page; with `449` tools, category filters and search are essential and present.

## Issues Found

- Many tools have thin FAQ coverage: `397` tools have fewer than two FAQs.
- `llms.txt` is stale and advertises `205+` tools instead of the current `449`.
- Some metadata lengths exceed common SEO display guidance: `2` long titles and `1` long descriptions.
- Several tool families carry policy, AdSense, brand, legal, or misuse risk and should not all be indexed at once.
- The generic tool page has a large inline script architecture, which is acceptable for launch validation but not ideal for long-term performance.

## Critical Fixes Required Before Launch

- No code-integrity blocker was found.
- Before broad SEO/AdSense launch, apply an indexing policy from `artifacts/index_priority_plan.csv`, especially for `noindex` and `low` tools.
- Update `public/llms.txt` tool count and category claims so public machine-readable documentation matches the current inventory.

## Recommended Fixes After Launch

- Add at least two unique FAQs to the `397` tools currently flagged as weak.
- Shorten the flagged meta title/meta description entries.
- Add or tighten disclaimers for legal, fake-document, security key/token, and sensitive cultural tools.
- Split the giant generator switch into per-tool modules or lazy-loaded logic to reduce page weight.
- Review direct franchise/platform tools and rewrite, noindex, or merge them into generic brand-safe modes.
- Add automated inventory checks to CI for duplicate slugs, broken related slugs, invalid categories, metadata length, and index-priority drift.
